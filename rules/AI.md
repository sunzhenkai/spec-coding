# 规约

这是给 AI 生成代码的规约文件。

# 代码

1. 注意代码结构，使用好的设计模式，但不要把问题复杂化
2. 注意错误打印的人类友好性，尽量简洁明确
3. 如果有更好的方案要及时跟用户确认
4. 按照功能模块分步骤实现，每次聚焦一个完整功能
5. 注意清晰的路线，先基础架构 → 再核心功能 → 最后优化
6. 测试用的脚本文件要及时清除

# 文档

1. 不要写过于冗余的文档，应尽量简洁明了
2. 注意文档组织方式，尽量和代码分开，且有层次、分门别类
3. Plan 也要保存在合适的位置

# 解决问题

1. 遇到错误时，先分析可能的原因
2. 对原因要进行必要的确认，模糊的地方及时向用户确认
3. 确认的原因后，尽量使用更优雅、简单的方式解决
4. 如果需要小范围重构及时向用户确认
