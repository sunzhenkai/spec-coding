# AI 代码生成规约

## 代码规范

1. 保持简洁，使用合适的设计模式, 但避免过度设计，优先选择简单直接的方案
2. 错误信息要清晰明确，便于理解和定位问题
3. 发现更好的实现方案时主动提出，让用户选择
4. 按模块分步实现，每步完成一个完整功能
5. 遵循开发顺序：基础架构 → 核心功能 → 优化完善
6. 测试脚本、文件及时删除
7. 代码要有适当的注释，关键逻辑必须说明
8. 遵循项目现有的代码风格和命名规范
9. 考虑边界情况和异常处理，避免潜在bug

## 文档规范

1. 文档要简洁，只写必要内容，避免废话
2. 文档与代码分离，按类别组织存放
3. 设计方案和计划文档统一保存在 docs/ 目录
4. 接口文档要包含示例，便于理解和使用
5. 更新代码时同步更新相关文档
6. spes/ 路径为人类规约文档，AI 只读不要修改

## 问题处理

1. 遇到错误先分析原因，不要盲目尝试
2. 模糊或不确定的地方及时向用户确认
3. 优先选择优雅简单的解决方案，避免复杂化
4. 需要重构时先说明原因和范围，征得用户同意
5. 记录遇到的问题和解决方案，便于后续参考

## 沟通协作

1. 理解需求后再动手，必要时先复述确认
2. 复杂任务提供实现思路，让用户了解方向
3. 及时报告进度和遇到的问题
4. 代码修改要说明改动内容和原因
5. 提供可运行、可测试的代码，确保质量
